# FFBayes Documentation

Welcome to the FFBayes documentation! This guide helps you find the right information for your needs.

Generated outputs are written to the configured runtime output directory during execution. Published copies are synced into cloud `data/` plus a dated `Analysis/<date>/` snapshot only when you run `ffbayes publish` (or `ffbayes-publish`).
The GitHub Pages dashboard is staged into `site/index.html` with `ffbayes publish-pages` (or `ffbayes-publish-pages`).
If you need a non-default runtime location, set `FFBAYES_RUNTIME_ROOT` explicitly.

## 🎯 **Quick Reference**

### **Essential Commands**
```bash
# Pre-draft pipeline
ffbayes pre-draft

# Draft board and decision backtest
ffbayes draft-strategy
ffbayes draft-backtest

# Explicit publication
ffbayes publish --year 2026

# GitHub Pages dashboard
ffbayes publish-pages --year 2026
```

### **Dashboard Notes**
- The “live dashboard” is a self-contained HTML artifact generated by `ffbayes draft-strategy`.
- For convenience, `ffbayes draft-strategy` stages a repo-root shortcut at `dashboard/index.html` (gitignored) plus the paired `dashboard/dashboard_payload.json`.
- For convenience, `ffbayes draft-strategy` also stages the same narrow shortcut pair under the configured runtime root at `dashboard/index.html` and `dashboard/dashboard_payload.json`.
- `site/index.html` is just the GitHub Pages site root. It stays a placeholder until you run `ffbayes publish-pages --year <year>` to copy the generated HTML into `site/index.html`.


### **Key Files**
- **Team File**: pass `--team-file <path>` (or set `TEAM_FILE`) for legacy team-roster analyses
- **Main Output**: `runs/<year>/pre_draft/artifacts/draft_strategy/draft_board_<year>.xlsx`
- **Dashboard Payload**: `runs/<year>/pre_draft/artifacts/draft_strategy/dashboard_payload_<year>.json`
- **HTML Dashboard**: `runs/<year>/pre_draft/artifacts/draft_strategy/draft_board_<year>.html`
- **Convenience Shortcut (repo root)**: `dashboard/index.html`
- **GitHub Pages Site**: `site/index.html`
- **Decision Backtest**: `runs/<year>/pre_draft/artifacts/draft_strategy/draft_decision_backtest_<year_range>.json`
- **Configuration**: JSON config files (see main README)

### **Common Issues**
- **Player Names**: Must match database exactly (e.g., "Patrick Mahomes" not "P. Mahomes")
- **File Format**: TSV with columns: POS, PLAYER, BYE
- **Environment**: Activate conda environment: `conda activate ffbayes`

## 🎉 **Getting Started**

1. **Read the [Main README](../README.md)** for complete setup instructions
2. **Check [Output Examples](OUTPUT_EXAMPLES.md)** to see what you'll get
3. **Run the draft helper**: `ffbayes draft-strategy`
4. **Open the generated dashboard** at `dashboard/index.html` or the matching runtime shortcut under the configured runtime root
5. **Use `site/index.html` only after `ffbayes publish-pages --year <year>`** if you want the GitHub Pages staging path
