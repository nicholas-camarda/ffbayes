# FFBayes Documentation

Welcome to the FFBayes documentation! This guide helps you find the right information for your needs.

Generated outputs are written to the configured runtime output directory during execution. Published copies are synced into cloud `data/` plus a dated `Analysis/<date>/` snapshot only when you run `ffbayes publish` (or `ffbayes-publish`).
The GitHub Pages dashboard is staged into `site/index.html` with `ffbayes publish-pages` (or `ffbayes-publish-pages`).
If you need a non-default runtime location, set `FFBAYES_RUNTIME_ROOT` explicitly.

## Quick Reference

### Essential Commands
```bash
# Full supported pre-draft workflow
ffbayes pre-draft

# Regenerate board artifacts from current processed inputs
ffbayes draft-strategy

# Restage Pages from current dashboard artifacts
ffbayes publish-pages --year 2026

# Import finalized draft downloads after the draft
ffbayes draft-retrospective --import-finalized ~/Downloads/ffbayes_finalized_*_2026_* --ingest-only --year 2026
```

### What To Open
- For local use after a run: `dashboard/index.html`
- For staged GitHub Pages output: `site/index.html`
- For the canonical runtime payload: `runs/<year>/pre_draft/artifacts/draft_strategy/dashboard_payload_<year>.json`

### Dashboard Notes
- The “live dashboard” is a self-contained HTML artifact generated by `ffbayes draft-strategy`.
- For convenience, `ffbayes draft-strategy` stages a repo-root shortcut at `dashboard/index.html` (gitignored) plus the paired `dashboard/dashboard_payload.json`.
- For convenience, `ffbayes draft-strategy` also stages the same narrow shortcut pair under the configured runtime root at `dashboard/index.html` and `dashboard/dashboard_payload.json`.
- `site/index.html` is the staged GitHub Pages copy, not the local authoritative working surface.
- `site/index.html` only changes after you run `ffbayes publish-pages --year <year>` and commit the updated `site/` files.

### Key Files
- Main output workbook: `runs/<year>/pre_draft/artifacts/draft_strategy/draft_board_<year>.xlsx`
- Dashboard payload: `runs/<year>/pre_draft/artifacts/draft_strategy/dashboard_payload_<year>.json`
- HTML dashboard: `runs/<year>/pre_draft/artifacts/draft_strategy/draft_board_<year>.html`
- Convenience shortcut: `dashboard/index.html`
- GitHub Pages staging root: `site/index.html`
- Decision backtest: `runs/<year>/pre_draft/artifacts/draft_strategy/draft_decision_backtest_<year_range>.json`
- Configuration: `config/user_config.json`

### Common Issues
- Activate the environment first: `conda activate ffbayes`
- Run `ffbayes pre-draft` before expecting `dashboard/index.html` to exist in a fresh clone
- Use `dashboard/index.html` for local drafting, not `site/index.html`
- If the dashboard HTML looks stale after template-only changes, use `ffbayes refresh-dashboard --year <year>`
